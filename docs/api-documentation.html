Predictive Maintenance API

Real-time machine failure prediction from sensor data

Base URL

Local: http://127.0.0.1:8000
Docker: http://localhost:8000
Interactive Docs: http://127.0.0.1:8000/docs

Endpoints

GET /

Health check - verifies API is running

POST /predict

Predict machine failure from sensor data

POST /predict

Accepts sensor telemetry and returns failure prediction with probability.

Request Parameters

Parameter Type Status Description
Air_temperature_K float Required Ambient air temperature in Kelvin
Process_temperature_K float Required Machine process temperature in Kelvin
Rotational_speed_rpm float Required Machine rotational speed in RPM
Torque_Nm float Required Machine torque in Newton-meters
Tool_wear_min float Required Tool wear in minutes
Type_L int Required Binary: 1 if Low quality, 0 otherwise
Type_M int Required Binary: 1 if Medium quality, 0 otherwise

Example Request

cURL

curl -X POST "http://127.0.0.1:8000/predict" \ -H "Content-Type: application/json" \ -d '{ "Air_temperature_K": 298.1, "Process_temperature_K": 308.6, "Rotational_speed_rpm": 1551, "Torque_Nm": 42.8, "Tool_wear_min": 0, "Type_L": 0, "Type_M": 1 }'

Python (requests)

import requests payload = { "Air_temperature_K": 298.1, "Process_temperature_K": 308.6, "Rotational_speed_rpm": 1551, "Torque_Nm": 42.8, "Tool_wear_min": 0, "Type_L": 0, "Type_M": 1 } response = requests.post( "http://127.0.0.1:8000/predict", json=payload ) print(response.json())

Response

200 OK

Successful prediction

{ "machine_status": "Healthy", "failure_probability": "0.00%", "sensor_data_processed": true }

Response Fields:

  • machine_status (string): "Healthy" or "Failure Predicted"
  • failure_probability (string): Percentage probability of failure (0-100%)
  • sensor_data_processed (boolean): Confirmation that data was processed

503 Service Unavailable

Model failed to load

{ "detail": "Model is not available. Check server logs for load errors." }

400 Bad Request

Invalid input format or missing fields

{ "detail": "Error message describing the validation failure" }

Model Details

Property Value
Algorithm Random Forest Classifier
N Estimators 100 trees
Class Imbalance Handling Balanced class weights
Feature Engineering Temp_diff_K, Power_W computed at inference
Precision (on test set) 91%
Avg Inference Time ~0.07 ms per prediction

Feature Engineering

The model computes domain-specific features at inference time:

Temp_diff_K = Process_temperature_K - Air_temperature_K Power_W = Rotational_speed_rpm × Torque_Nm

These engineered features are computed at inference time from raw sensor inputs.
